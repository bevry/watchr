Watcher

./source/index.js

Watcher Watches a path and if its a directory, its children too, and emits change events for updates, deletions, and creations

Available events:

  • log(logLevel, ...args) - emitted for debugging, child events are bubbled up
  • close(reason) - the watcher has been closed, perhaps for a reason
  • change('update', fullPath, currentStat, previousStat) - an update event has occured on the fullPath
  • change('delete', fullPath, currentStat) - an delete event has occured on the fullPath
  • change('create', fullPath, null, previousStat) - a create event has occured on the fullPath

Parameters
opts (Object) the configuration to use
Name Description
opts.ignoreCustomPatterns [RegExp] A regular expression that if matched again the path will ignore the path
opts.interval [number] (default 5007) If the watchFile method was used, this is the interval to use for change detection if polling is needed
opts.persistent [boolean] (default true) If the watchFile method was used, this is whether or not watching should keep the node process alive while active
opts.catchupDelay [number] (default 2000) This is the delay to wait after a change event to be able to detect swap file changes accurately (without a delay, swap files trigger a delete and creation event, with a delay they trigger a single update event)
opts.stat [Stats] A stat object for the path if we already have one, otherwise it will be fetched for us
opts.followLinks [boolean] (default true) If true, will use fs.stat instead of fs.lstat
opts.ignorePaths [Array<string>] (default false) Array of paths that should be ignored
opts.ignoreHiddenFiles [boolean] (default false) Whether to ignore files and directories that begin with a .
opts.ignoreCommonPatterns [boolean] (default false) Whether to ignore common undesirable paths (e.g. .svn , .git , .DS_Store , thumbs.db , etc)
opts.preferredMethods [Array<string>] (default ['watch','watchFile']) The order of watch methods to attempt, if the first fails, move onto the second
